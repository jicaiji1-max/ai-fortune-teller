## 4. 数据模型设计（第 4 轮）

### 核心数据结构

```typescript
interface AppData {
  transactions: Transaction[];  // 交易记录
  accounts: Account[];          // 账户
  categories: Category[];       // 分类
  settings: Settings;           // 设置
  meta: Meta;                   // 元数据
}
```

### 关键设计决策

1. **金额存储**: 使用「分」为单位（避免浮点误差）
2. **日期格式**: ISO 字符串 YYYY-MM-DD
3. **ID 生成**: UUID v4
4. **存储方案**: localStorage（key: bookkeeping_app_data）
5. **索引优化**: 按日期建立索引

### API 服务层设计

- TransactionService（交易服务）
- AccountService（账户服务）
- CategoryService（分类服务）
- ExportService（导出服务）
