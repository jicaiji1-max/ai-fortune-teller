## 技术风险识别与解决方案

### 1. 数据丢失风险 🔴
- **风险**: localStorage 可能被浏览器清理
- **概率**: 中
- **影响**: 高
- **解决方案**: 
  - 提供数据导出功能（JSON + CSV）
  - 定期备份提醒（每 7 天）
  - 浏览器存储状态检测

### 2. 大数据量性能 🟡
- **风险**: 超过 1000 条记录可能卡顿
- **概率**: 中
- **影响**: 中
- **解决方案**: 
  - 虚拟列表（react-window）
  - 分页加载（每页 50 条）
  - 索引优化（按日期索引）

### 3. 跨设备同步 🔴
- **风险**: 纯 localStorage 无法同步
- **概率**: 高
- **影响**: 中
- **解决方案**: 
  - MVP 不做（明确告知用户）
  - 后续版本加云端同步（IndexedDB + 后端）

### 4. 浏览器兼容性 🟡
- **风险**: localStorage 在隐私模式下不可用
- **概率**: 低
- **影响**: 中
- **解决方案**: 
  - 添加存储可用性检测
  - 降级提示（建议使用正常模式）

## 技术实现方案确认

### ✅ 快速记账功能
- 首页悬浮按钮（FAB）
- 弹窗表单（Modal）
- 3 步完成：选类型 → 输金额 → 确认
- 默认当天，支持修改日期

### ✅ CSV 导出功能
- 使用 PapaParse 库
- 支持 UTF-8 BOM（Excel 兼容）
- 导出字段：日期、类型、金额、分类、账户、备注

### ✅ 基础报表功能
- 仅做月度收支对比柱状图
- 使用 Recharts 库
- 支持切换月份

### ✅ 响应式设计
- 一套代码
- CSS Grid + Flexbox
- 断点：768px（平板）、375px（手机）

## 技术决策

✅ 不做 ML 自动分类（复杂度太高）
✅ 不做语音记账（兼容性差）
✅ 不做多账本（MVP 聚焦）
✅ 必须做数据导出（降低丢失风险）
