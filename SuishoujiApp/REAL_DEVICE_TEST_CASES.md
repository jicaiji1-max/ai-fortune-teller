## 🔴 Critical 测试用例

### TC-REAL-001: 删除带图片的笔记

**优先级：** P0 - Critical

**发现时间：** 2026-03-11 00:58

**问题描述：**
- 文字笔记可以删除 ✅
- 带图片的笔记删除不了 ❌

**测试步骤：**
1. 创建一条纯文字笔记
2. 点击删除按钮 → 确认删除
3. 验证笔记已删除 ✅

4. 创建一条带图片的笔记（从相册或拍照）
5. 点击删除按钮 → 确认删除
6. 验证笔记已删除 ❌（失败）

**根本原因：**
- `try?` 吞掉了错误，无法看到失败原因
- SwiftData 删除大对象可能需要特殊处理

**修复方案（已实施）：**
```swift
// 1. 改用 do-catch 捕获错误
do {
    // 2. 带图片的笔记先清空数据
    if note.photoData != nil {
        note.photoData = nil
        note.thumbnailData = nil
        try modelContext.save()
    }
    
    // 3. 然后删除
    modelContext.delete(note)
    try modelContext.save()
} catch {
    // 4. 显示错误信息
    showDeleteError = true
}
```

**验证方式：**
- 真机测试（手动）
- 检查错误日志

**预期结果：**
- ✅ 带图片的笔记能正常删除
- ✅ 如果失败，显示错误信息

### TC-REAL-002: 拍照后添加笔记

**优先级：** P0 - Critical

**发现时间：** 2026-03-11 00:58

**问题描述：**
- 从相册选择图片可以添加 ✅
- 拍照后无法添加 ❌

**测试步骤：**
1. 点击 + 按钮 → 选择"拍照"
2. 拍摄一张照片 → 使用照片
3. 添加说明（可选）
4. 点击保存
5. 验证笔记已添加到列表 ❌（失败）

**根本原因：**
- `image.jpegData(compressionQuality: 1.0)` 可能返回 nil
- 或者原图太大导致内存问题

**修复方案（已实施）：**
```swift
// 降低初始质量，添加降级策略
if let data = image.jpegData(compressionQuality: 0.9) {
    selectedImageData = data
} else {
    // 失败时尝试更低质量
    selectedImageData = image.jpegData(compressionQuality: 0.7)
}
```

**验证方式：**
- 真机拍照测试
- 不同光线和场景

**预期结果：**
- ✅ 拍照后能正常添加笔记
- ✅ 图片质量可接受

### TC-REAL-003: 图片清晰度

**优先级：** P1 - High

**发现时间：** 2026-03-11 00:58

**问题描述：**
- 图片第一次加载时模糊 ❌
- 后续加载才清晰 ✅

**测试步骤：**
1. 添加一张高清图片（从相册）
2. 立即查看列表中的显示效果
3. 观察图片是否清晰

**根本原因：**
- 缩略图使用 `preparingThumbnail(of: 200x200)` 缩小了
- 但 UI frame height=200，宽度不限
- SwiftUI 放大 200x200 缩略图导致模糊

**修复方案（已实施）：**
```swift
// 缩略图直接使用，不再 preparingThumbnail
if let data = thumbnailData {
    return UIImage(data: data)
}

// 原图提高尺寸到 600x600
if let data = photoData, let image = UIImage(data: data) {
    return image.preparingThumbnail(of: CGSize(width: 600, height: 600))
}
```

**验证方式：**
- 真机测试，添加高清图片
- 观察第一次加载的清晰度

**预期结果：**
- ✅ 图片第一次加载就清晰
- ✅ 滚动流畅（性能不受影响）
