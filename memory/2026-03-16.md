## 修复记录（2026-03-16）

### 1. 相册黑屏（CustomPhotoPickerView）
- **根因**：iOS 26 beta 的 `NavigationStack` 在 sheet 里不渲染
- **修法**：去掉 NavigationStack，自绘 header bar（ZStack + HStack）

### 2. 图片预览黑屏（PhotoViewerView）
- **根因**：iOS 26 beta sheet-in-sheet / fullScreenCover 渲染 bug
- **修法**：完全用 UIKit 重写 `GalleryViewController`（外层分页 ScrollView + 内层 ZoomScrollView），通过 `PreviewManager.shared.presentOnTop()` 直接 present 到 UIWindow 顶层

### 3. 视频预览黑屏
- **根因**：AVPlayerLayer 需要 CALayer frame 确定后才能渲染
- **修法**：等 400ms（fullScreenCover 动画完成）再显示播放器

### 4. 预览下滑关闭把编辑框带走
- **根因**：UIKit pan 手势移动 `self.view`，在 overlay 场景下影响父视图
- **修法**：只移动 `pageVC.view`（内容层），不动根视图；改用 `PreviewManager` present 到 UIWindow 顶层后彻底绕开

### 5. 图片+视频混合预览无法翻页
- **根因**：ZoomScrollView 的 panGesture 拦截横向滑动
- **修法**：`isScrollEnabled = false`（未放大时），放大后才开启，让分页 ScrollView 自由翻页

### 6. 预览关闭后按钮无法点击
- **根因**：`GalleryViewController` 关闭时只调 `onDismiss()`，没有 `dismiss(animated:)`，VC 残留拦截触摸
- **修法**：`close()` 改为 `dismiss(animated: true) { onDismiss?() }`

### 7. stableCopy 扩展名 bug
- **根因**：相册视频是 `.mov`，但 stableCopy 强制写 `.mp4`
- **修法**：`let ext = url.pathExtension.isEmpty ? "mp4" : url.pathExtension`

### 8. 视频路径重装后失效
- **根因**：存绝对路径，重装后 Container ID 变化
- **修法**：存相对路径（相对 Documents），`Note.videoURLs` 读时动态拼接当前 Documents 目录

### 9. 重装后视频文件消失
- **根因**：重装清空 Documents/Videos/
- **修法**：ContentView 启动时调 `restoreMissingVideos()`，用 `videoAssetIds` 的 PHAsset ID 从相册重新导出

    static let shared = PreviewManager()
    func present(items: [MediaPreviewItem], index: Int)  // 混合图片+视频

    func presentVideos(_ urls: [URL], index: Int)
    // 通过 presentOnTop() 直接 present 到 UIWindow key window
}

enum MediaPreviewItem {
    case photo(UIImage)
    case video(URL)
}

## 今日教训

1. **iOS 26 beta 的 SwiftUI modal（sheet/fullScreenCover）在嵌套场景不可靠**，遇黑屏优先 UIKit 或 UIWindow presentOnTop
2. **视频不能存 Documents 绝对路径**，重装必丢，要存相对路径 + assetId 双保险
3. SwiftUI `LazyVStack(pinnedViews:)` 的 sticky header 不靠谱，改用 VStack 把固定内容放 ScrollView 外
4. `@Environment(\.isSearching)` 只在 `.searchable` 的子视图内有效，不能在 parent 读
