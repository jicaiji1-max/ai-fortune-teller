### 问题诊断（01:45-01:53）

#### 问题 1：删除按钮点不了（进入编辑页面）

**症状：**
- 点击删除按钮 → 进入编辑页面
- 删除功能无法触发

**根本原因：**
```swift
// ContentView.swift 第 92-96 行
NoteRow(note: note) { ... }
.onTapGesture {  // ❌ 捕获了整个卡片的点击
    editingNote = note
    showEditSheet = true
}
```

**技术原因：**
- `.onTapGesture` 作用在整个 NoteRow 上
- 点击删除按钮时，事件被外层的 tap 手势捕获
- SwiftUI 的手势优先级问题

#### 问题 2：拍照功能不工作

**症状（详细询问后确认）：**
- 点击"拍照"按钮 → 相机打开 ✅
- 拍照完成 → 点"Use Photo" ✅
- **返回后图片不显示** ❌
- 相册选择功能正常 ✅

**根本原因：**
- `UIImagePickerControllerDelegate` 回调可能在后台线程
- `@State` 更新在非主线程不可靠
- `dismiss` 时序导致状态丢失

### 修复过程（01:51-01:54）

#### 修复 1：删除按钮（事件冲突）✅

**方案选择：**
- 方案 1：移除 onTapGesture，给 NoteRow 添加编辑回调（✅ 采用）
- 方案 2：让删除按钮阻止事件传播

**修改代码：**

**NoteRow.swift：**
```swift
struct NoteRow: View {
    let note: Note
    let onDelete: () -> Void
    let onEdit: () -> Void    // ← 新增编辑回调
    
    var body: some View {
        VStack(...) { ... }
        .contentShape(Rectangle())  // ← 明确点击区域
        .onTapGesture {
            onEdit()  // ← 在内部处理点击
        }
    }
}
```

**ContentView.swift：**
```swift
NoteRow(note: note, onDelete: {
    noteToDelete = note
    showDeleteAlert = true
}, onEdit: {
    editingNote = note
    showEditSheet = true
})
// ← 移除外层 .onTapGesture
```

**编译结果：** ✅ BUILD SUCCEEDED

### 技术收获

**1. SwiftUI 手势优先级**
- Button 内置手势优先级最高
- 父视图的 `.onTapGesture` 不会捕获 Button 点击
- 但最好的做法是将逻辑封装在组件内部

**2. UIKit 回调的线程安全**
- `UIImagePickerControllerDelegate` 回调在主线程
- 但仍建议显式用 `DispatchQueue.main.async`
- 避免依赖未文档化的行为

**3. 状态更新时序**
- SwiftUI 的 `@State` 更新必须在主线程
- dismiss 操作可能清空上下文，需要先完成回调
- 延迟 dismiss 是常见模式

**4. Claude Code 的价值**
- 作为第二双眼睛审查代码
- 发现潜在的改进点
- 确认修复的正确性
