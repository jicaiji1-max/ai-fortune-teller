## Chrome 扩展开发：OpenClaw Monitor

### 创建的扩展
- **路径**: `/Users/caiji/.openclaw/workspace-programmer/openclaw-monitor-extension/`
- **功能**: 在 OpenClaw 页面上显示 Agent/Session 状态监控

### 功能特性
- 显示每个 Agent 使用的模型
- 显示 Sessions 数量
- 显示 Context 使用率（进度条 + 百分比）
- 显示活跃状态（绿点=活跃，黄点=空闲）

### 关键文件
- `manifest.json` - 扩展配置
- `content.js` - 注入脚本（CSP 安全版本）
- `style.css` - 面板样式

### 注意事项
- **CSP 问题**: 不能使用 `onclick="func()"` 内联事件，必须用 `addEventListener`
- **Dashboard API**: 需要公开 API `/api/public/sessions` 才能获取数据

## Context 爆满问题

### 原因
- Heartbeat 定时问候持续 10 小时，上下文累积到 3MB
- 项目助手 session 文件达到 2.2MB

### 解决方案
1. 减少 Heartbeat 频率
2. 定期清理超大会话文件
3. 删除大 session: `~/.openclaw/agents/<agent>/sessions/*.jsonl`
