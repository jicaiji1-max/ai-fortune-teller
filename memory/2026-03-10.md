### 技术分析（23:50-23:59）

#### Claude Code 的认证机制

**之前查到的认证信息：**
```json
{
  "authMethod": "claude.ai",
  "apiProvider": "firstParty",
  "apiKeySource": "/login managed key",
  "email": "jicaiji1@gmail.com"
}
```

**关键发现：**
- Claude Code 使用 **firstParty 认证**
- 不是标准的外部 API key
- 是通过 claude.ai 账户登录获得的内部 key
- 由 Anthropic 管理，不是用户生成的

#### firstParty vs 外部 API Key

**firstParty managed key：**
- ✅ 需要 claude.ai 账户登录
- ✅ 由 Anthropic 内部管理
- ✅ 绑定到登录会话
- ✅ 在登录的环境中自动可用
- ❌ 不能作为环境变量直接使用
- ❌ 缺少认证上下文时无效

**外部 API Key：**
- ✅ 从 console.anthropic.com 生成
- ✅ 可以直接作为 `x-api-key` header 使用
- ✅ 不需要登录会话
- ✅ 可以在任何环境中使用
