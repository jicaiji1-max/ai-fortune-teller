### 今日完成的主要工作

#### Bug 修复
1. **图片添加不稳定** - 重构 CameraView，简化视图表达式，修复 PhotosPicker 加载逻辑
2. **编辑时自动更新时间** - 改为只在内容实际变化时更新 timestamp（比较 text/photo/type/location）
3. **相机崩溃 (SIGKILL)** - 根因：Info.plist 缺少 `NSLocationWhenInUseUsageDescription`，导致 dyld 启动阶段被 kill
4. **SF Symbol 兼容性** - `camera.badge.plus` 是 iOS 17+ 才有，改为 `photo.on.rectangle`（iOS 14+ 可用）
5. **ImagePicker dismiss 崩溃** - 改为 `picker.dismiss(animated: true, completion: nil)` 而非 `parent.dismiss()`
6. **图片编辑按钮点不动** - 根因：overlay 中 Button 消费点击事件，PhotosPicker 不弹出。改为给"添加图片"按钮独立绑定 PhotosPicker
7. **Invalid frame dimension 警告** - 网格图片缺少宽度约束，加 `frame(minWidth: 0, maxWidth: .infinity)` + `.clipped()`

#### 代码质量修复
- `[weak self]` 用于 struct（编译错误）→ 改为直接用 `self`
- `self?.` 残留（CameraView + TextEditorView）→ 改为 `self.`
- `LocationFetcher` 局部变量被 ARC 提前释放 → 用 `@State private var locationFetcher: AnyObject?` 强持有
- `import CoreLocation` 缺失（CameraView）、多余（Note.swift）→ 修正
- `ForEach(0..<count)` 越界风险 → 改为 `Array(photoLibrary.enumerated())`
- `photoLibrary.first!` 强制解包 → 改为 `guard let` 保护
- `PHPhotoLibrary.requestAuthorization` 废弃 API → 改为 `.requestAuthorization(for: .addOnly)`
- Task 异步竞争：`isAppendMode` 在 Task 外捕获快照值 `let shouldAppend = isEditMode`

#### 新功能实现
1. **位置记录功能** - Note 模型添加 locationName/latitude/longitude 字段；CameraView 和 TextEditorView 添加位置获取按钮；NoteRow 显示位置信息
2. **照片同步到系统相册** - 拍照后自动保存到相册（仅新建模式，编辑模式不重复保存）；Info.plist 添加 `NSPhotoLibraryAddUsageDescription`
3. **编辑模式图片网格** - 3列网格展示，每张可删（至少保留1张），网格末尾有＋添加按钮；新建模式保持横向滚动
