## Chrome 扩展监控面板 - 已完成 ✅

### 问题根因
- **CORS 问题**: Dashboard 的公开 API 没有正确设置 `Access-Control-Allow-Origin` 头
- Chrome 扩展从 `localhost:18789` 访问 `localhost:7070` 时被浏览器拦截

### 解决方案
在 Dashboard 的 `/api/public/sessions` 端点添加了正确的 CORS 头：
```
Access-Control-Allow-Origin: *
Access-Control-Allow-Methods: GET, OPTIONS
Access-Control-Allow-Headers: Content-Type
```

- 显示活跃状态（绿点=活跃，黄点=空闲）
- 每 10 秒自动刷新

## Dashboard 只读取 main agent 的 sessions 问题

### 问题发现
Dashboard 的 `getSessionsJson()` 函数只读取 `~/.openclaw/agents/main/sessions`，没有读取其他 agent 的 sessions 数据。

### 修复方案
修改 `getSessionsJson()` 函数，遍历 `~/.openclaw/agents/` 下所有 agent 的 sessions 目录，读取所有 agent 的 sessions.json 文件。

### 注意事项
- 修改 Dashboard 代码后需要重启 Dashboard
- `getLastMessage()` 和 `getSessionCost()` 函数可能需要额外的 agentSessDir 参数
